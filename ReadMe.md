# TinaBorke.Art

Сайт визажиста-гримера Тины Борке на FastAPI. Проект объединяет публичный сайт, форму записи, Telegram-CRM уведомления, SQLite-хранилище контента, SEO-страницы, блог и защищенную админ-панель.

## Что умеет проект

### Публичный сайт

- Главная страница `/` с hero-блоком, акцией, описанием мастера, услугами, отзывами, контактами и формой записи.
- Страница о мастере `/about`.
- Страницы услуг с ЧПУ: `/uslugi/{slug}`.
- Блог `/blog`.
- Страница поста `/blog/{slug}`.
- Health-check endpoint `/health`.
- Статические файлы из `static/`: CSS, JS, изображения, favicon.

### Запись клиентов

- Основная форма отправляет заявку на `/api/booking`.
- Быстрая форма отправляет заявку на `/api/quick-booking`.
- Данные заявки сохраняются в SQLite-таблицу `bookings`.
- Поля заявки: имя, телефон, услуга, дата, сообщение, время создания, статус.
- Телефон и имя валидируются через Pydantic.
- После создания заявки запускается фоновая отправка уведомления в Telegram.

### Telegram-CRM

- Уведомления о новых заявках отправляются администратору и сотрудникам.
- Используются переменные `TELEGRAM_BOT_TOKEN`, `ADMIN_TELEGRAM_ID`, `STAFF_TELEGRAM_IDS`.
- Токен не выводится в лог целиком.
- Если Telegram недоступен, заявка все равно сохраняется в базе, а ошибка логируется.

### Админ-панель

Админка доступна по адресу:

```text
/admin
```

Защита: HTTP Basic Auth.

В админке можно редактировать:

- имя мастера;
- город;
- телефон;
- Telegram-ссылку;
- часы работы;
- районы выезда;
- текст «О себе»;
- текст акции на главной;
- SEO title и description главной;
- SEO title и description блога;
- услуги;
- отзывы;
- портфолио;
- блог-посты.

### Управление услугами

Для каждой услуги доступны:

- название;
- slug для ЧПУ;
- цена;
- время выполнения;
- популярная / основная / дополнительная группа;
- флаг «Хит»;
- описание;

- SEO title;
- SEO description;
- флаг видимости;
- добавление;

Активные услуги отображаются:

- на главной странице;
- в форме записи;
- в sitemap;
- на отдельных страницах `/uslugi/{slug}`.

### Портфолио

Отдельный интерфейс «Галерея» убран, чтобы не дублировать портфолио. Все работы загружаются и редактируются через вкладку `Портфолио`.

Админка поддерживает загрузку изображений в:

```text
static/uploads/portfolio/
```

Проверяются расширения:

- `.jpg`
- `.jpeg`
- `.png`
- `.webp`

Опасные расширения вроде `.php`, `.html`, `.js`, `.svg`, `.exe` не принимаются.

Для изображений можно задать:

- alt-текст;
- раздел портфолио;
- связанную услугу;
- порядок сортировки;
- видимость.

### Отзывы

Отзывы хранятся в SQLite и могут редактироваться из админки.

Для отзыва доступны:

- имя клиента;
- текст;
- дата;
- флаг видимости;
- добавление;
- редактирование;
- удаление.

Активные отзывы отображаются на главной странице.

### Импорт из Telegram

В админке есть кнопка ручного запуска импорта:

```text
Запустить импорт Telegram сейчас
```

Также при старте приложения включается автоимпорт каждые 10 минут, если заданы:

```env
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHANNEL_ID=
```

Важно: Telegram Bot API не отдает старую историю канала. Импорт работает только с `channel_post` updates, которые бот реально получает после добавления в канал.

## Technical SEO

### Sitemap

Sitemap доступен по адресу:

```text
/sitemap.xml
```

Он формируется динамически из SQLite-базы и настроек админки. В него попадают только публичные страницы:

- главная `/`;
- страница о мастере `/about`;
- портфолио `/portfolio`;
- активные и не удаленные разделы портфолио `/portfolio/{category_slug}`;
- блог `/blog`;
- видимые и не удаленные посты `/blog/{slug}`;
- активные услуги `/uslugi/{slug}`.

В sitemap не добавляются `/admin`, `/api`, `/static`, скрытые услуги, скрытые или удаленные посты.

### Robots.txt

Robots доступен по адресу:

```text
/robots.txt
```

Он закрывает служебные разделы:

```text
Disallow: /admin
Disallow: /login
Disallow: /logout
```

И содержит ссылку на sitemap:

```text
Sitemap: https://example.ru/sitemap.xml
```

### BASE_URL

На продакшене обязательно задайте настоящий HTTPS-домен:

```env
BASE_URL=https://example.ru
```

Если `BASE_URL` не задан, приложение строит абсолютные URL от host текущего запроса. Для продакшена лучше не полагаться на fallback, чтобы canonical, sitemap, robots и Open Graph всегда указывали на основной домен.

### Title и Description

SEO title и description настраиваются:

- главная: вкладка админки с SEO-полями главной;
- блог: SEO-поля блога в админке;
- услуги: поля «Заголовок для поиска» и «Описание для поиска» в карточке услуги;
- посты блога: автоматически из текста поста, если ручные SEO-поля пустые;
- портфолио и категории: из названия и описания разделов.

Если поле пустое, сайт формирует безопасный fallback, чтобы не выводить пустой `<title>` или пустой `meta description`.

### Canonical

Canonical добавлен на публичные страницы:

- `/`;
- `/about`;
- `/portfolio`;
- `/portfolio/{category_slug}`;
- `/blog`;
- `/blog/{slug}`;
- `/uslugi/{slug}`.

Проверка вручную: открыть HTML страницы и найти:

```html
<link rel="canonical" href="https://example.ru/...">
```

### Open Graph

Open Graph формируется для публичных страниц:

```html
<meta property="og:title" content="...">
<meta property="og:description" content="...">
<meta property="og:type" content="website или article">
<meta property="og:url" content="...">
<meta property="og:site_name" content="Визаж & Грим от Тины Борке">
```

`og:image` выводится только если есть корректная абсолютная ссылка на существующее изображение.

Логика выбора изображения:

- главная: главное изображение сайта или favicon fallback;
- услуга: первое фото связанного раздела портфолио, иначе fallback сайта;
- портфолио: первое активное фото;
- категория портфолио: первое активное фото категории;
- пост блога: первое фото поста.

### Schema.org / JSON-LD

Используемые типы:

- главная: `BeautySalon` / `LocalBusiness`;
- услуга: `Service`;
- услуга с FAQ: дополнительно `FAQPage`;
- вложенные страницы: `BreadcrumbList`;
- пост блога: `BlogPosting`;
- портфолио и категории: `ImageObject`.

JSON-LD собирается на сервере, очищается от пустых полей и выводится только валидным JSON.

### Как проверить

1. Открыть `/sitemap.xml` и проверить, что URL абсолютные.
2. Убедиться, что в sitemap нет `/admin`.
3. Открыть `/robots.txt` и проверить ссылку `Sitemap`.
4. Открыть главную, услугу, пост и категорию портфолио через «Просмотр кода страницы».
5. Проверить `title`, `meta description`, canonical и Open Graph.
6. Скопировать JSON-LD в валидатор Schema.org.
7. На продакшене проверить, что sitemap подтягивает все данные из реальной базы, включая созданные через админку услуги, посты и разделы портфолио.

### Адаптивная верстка

В `static/css/style.css` добавлены брейкпоинты:

```css
@media (max-width: 640px) {}
@media (min-width: 641px) and (max-width: 1024px) {}
@media (min-width: 1025px) {}
```

Также добавлено:

- гибкая ширина контейнеров;
- адаптивные сетки услуг, отзывов, контактов, блога;
- минимальная высота кликабельных элементов `44px`;
- мобильная навигация;
- горизонтальный свайп галереи поста на мобильных;
- адаптивные формы.

### Производительность

- Для статики добавляется `Cache-Control: public, max-age=604800`.
- Изображения в новых шаблонах используют `loading="lazy"`, где это уместно.
- Не добавлены тяжелые frontend-библиотеки.
- Основной JS остается в `static/js/app.js`.

### Логирование

- Логи пишутся в консоль и в `app.log`.
- Есть formatter, который убирает emoji из консольного вывода, чтобы снизить риск проблем с кодировкой.
- Traceback не показывается пользователю, общие ошибки возвращают JSON с кодом 500.

## Telegram import setup

Импорт постов из Telegram запускается из админки: `/admin?tab=telegram`.

1. Создайте бота через `@BotFather` или используйте существующего.
2. Добавьте бота в Telegram-канал.
3. Выдайте боту права администратора или права видеть посты канала.
4. Заполните `.env`:

```env
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHANNEL_ID=
TELEGRAM_API_ID=
TELEGRAM_API_HASH=
TELEGRAM_SESSION_NAME=tinaborke
TELEGRAM_IMPORT_MODE=bot_api
```

5. Запустите сайт:

```powershell
python app.py
```

6. Откройте `/admin?tab=telegram` и нажмите `Запустить импорт сейчас`.

Что импортируется:

- строка `Заголовок:` используется как заголовок публикации;
- строка `Рубрика:` используется как рубрика публикации;
- блок после `Текст:` сохраняется как основной текст;
- если `Текст:` не указан, используется оставшийся текст сообщения;
- если заголовка нет или текст короче 300 символов, публикация сохраняется как черновик и не индексируется;
- фото из Telegram-сообщения скачиваются в `static/blog_photos/` и прикрепляются к посту;
- если Telegram прислал несколько фото одной медиагруппой, они привязываются к одному посту;
- первое фото используется как обложка, превью и `og:image`, если отдельная обложка не задана;
- если фото нет, создается обычный текстовый пост без ошибки.

Режимы:

- `TELEGRAM_IMPORT_MODE=bot_api` используется текущей реализацией через Telegram Bot API.
- `TELEGRAM_IMPORT_MODE=telethon` зарезервирован для исторического импорта; для него нужны `TELEGRAM_API_ID`, `TELEGRAM_API_HASH` и отдельная Telethon-сессия.

Важно про Bot API:

- бот не получает старую историю канала;
- бот увидит только новые `channel_post` после добавления в канал;
- если нужен импорт старых публикаций, нужен отдельный Telethon-скрипт;
- токен бота не хранится в коде и не выводится в логах.

Диагностика в админке показывает:

- режим импорта;
- задан ли токен;
- задан ли ID канала;
- последний запуск;
- сколько постов импортировано;
- последнюю ошибку.

Если импорт не работает:

- проверьте `TELEGRAM_BOT_TOKEN`;
- проверьте `TELEGRAM_CHANNEL_ID`;
- проверьте, что бот добавлен в канал;
- проверьте права администратора бота;
- проверьте логи приложения `app.log`;
- откройте `/admin?tab=telegram` и посмотрите поле последней ошибки импорта;
- проверьте, что канал публичный или ID указан корректно;
- если фото не прикрепляются, проверьте, что бот видит именно новые посты канала, файл не больше 8 МБ и Telegram Bot API возвращает `file_path`;
- для старых постов используйте Telethon.

Полезные команды:

```powershell
python check_database.py
python app.py
```

Если будет добавлен отдельный исторический импорт, его удобно оформить командой:

```powershell
python import_telegram_posts.py
```

## Ограничения и замечания

- `docker-compose.yml` сейчас выглядит как черновик под другую структуру (`backend`, `frontend`, `nginx.conf`). Для текущей структуры проекта основной способ запуска: `python app.py`.
- Telegram Bot API не импортирует старые посты канала. Для полноценного исторического импорта понадобится отдельная интеграция, например Telethon, и дополнительные учетные данные.
- Конвертация изображений в WebP пока не выполняется автоматически: сайт принимает WebP и обычные JPG/PNG.
- CSRF-защита для форм админки пока не добавлена; Basic Auth закрывает доступ, но для публичного продакшена CSRF лучше добавить отдельно.
- PageSpeed 80+ требует проверки после деплоя на реальном домене, с реальными изображениями и настройками сервера.

## Быстрый чек-лист перед публикацией

- Задать надежный `ADMIN_PASSWORD`.
- Задать настоящий `BASE_URL` с HTTPS-доменом.
- Проверить `TELEGRAM_BOT_TOKEN`, `ADMIN_TELEGRAM_ID`, `STAFF_TELEGRAM_IDS`.
- Добавить бота в Telegram-канал, если нужен импорт блога.
- Проверить `/sitemap.xml` и `/robots.txt`.
- Проверить мобильную версию в DevTools.
- Проверить форму заявки и получение Telegram-уведомления.
- Сделать резервную копию `tinaborke.db`.
